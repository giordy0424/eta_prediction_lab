# ETA Prediction for Last-Mile Logistics

Travel time prediction for last-mile logistics using open data (OpenStreetMap) and Machine Learning.

**Goal —** Overcome the economic barrier of commercial traffic APIs (e.g. Google Maps: ~$10,950 for a 1500×1500 matrix) by building an adjacency-matrix inference system based on open-source resources.

**Approach —** The NYC Taxi Trip Duration dataset (Kaggle, 1.22M Manhattan rides) is enriched with OSRM estimates, topological features (turns, intersections), and temporal encoding. A naive model (MLP) reproduces OSRM from coordinates alone (R²0.95). Residual architectures (DeepResMLP, R²=0.558 on the gap) correct the discrepancy between static estimates and actual duration.

**Inspiration —** Boeing et al. (2026) "Travel time prediction from sparse open data", showing that OSM features alone (traffic lights, intersections, turns) can generate accurate ETAs without historical traffic data.

**Data:** NYC Taxi Trip Duration (Kaggle), filtered to Manhattan island.
**Pipeline:** OSRM routing server → data enrichment → ML models → evaluation.

## Project Structure

```
├── data/                     Input: train.csv (Kaggle); output: ultimate_{train,test}.csv
├── models/                   Pre-trained .pth files (checked-in)
│   ├── naive/                Baseline MLP models (v1, v2, v3)
│   └── final/                Correction models (ResMLP, DeepResMLP, …)
├── osrm_data/                OSM extract + OSRM graph files (generated by Phase 1)
├── phase1_osrm/              Docker-based OSRM server
│   └── setup_server.sh       Extract, contract & run OSRM
├── phase2_dataset/           Data pipeline
│   ├── prepare_dataset.py    Manhattan filter + temporal features
│   ├── enrich_and_split.py   OSRM enrichment (map-matching, topology) + 80/20 split
│   └── mask.geojson          Manhattan polygon
├── phase3_training/          Model training (optional, hours)
│   ├── models.py             Architectures (ResMLP, DeepResMLP, MultiBranch, …)
│   ├── train_naive.py        Baseline MLP (predicts osrm_eta from coordinates)
│   └── train_final.py        Correction models (predicts gap = duration − osrm_eta)
├── phase4_evaluation/
│   └── evaluation.ipynb      Jupyter notebook with scatter plots and metrics
├── run_pipeline.sh           Orchestration script (--phase1/2/3/4/--all)
├── requirements.txt
└── .gitignore
```

## Quick Start

```bash
# 1. Dependencies
pip install -r requirements.txt

# 2. Data — download and place:
#    - Kaggle (www.kaggle.com/competitions/nyc-taxi-trip-duration/data) train.csv         → data/train.csv
#    - OSM extract (extract.bbbike.org)     → osrm_data/manhattan.osm.pbf

# 3. Start OSRM server (Docker, leave it running)
#    Produces: osrm_data/*.osrm, *.osrm.names, *.osrm.tree, etc. (OSRM graph files)
#    Starts a Docker container serving the routing API on port 5000.
./run_pipeline.sh --phase1

# 4. Generate enriched dataset (Manhattan filter + OSRM + 80/20 split)
#    Phase 2 runs two scripts:
#      (a) prepare_dataset.py  →  data/_prepared.csv  (intermediate, auto-deleted)
#      (b) enrich_and_split.py →  data/ultimate_train.csv  (80% training set)
#                              →  data/ultimate_test.csv   (20% test set)
#    Optionally produces:  errors_v2.csv  (if OSRM requests fail)
./run_pipeline.sh --phase2

# 5. (Optional, takes hours) Train models from scratch
#    Produces:
#      models/naive/model_mlp_v1.pth, model_mlp_v2.pth, model_mlp_v3.pth
#      models/final/model_mlp_v2.pth, customModel1.pth, model_resmlp.pth,
#                 model_multibranch.pth, model_multibranch_resnet.pth,
#                 model_deep_resmlp.pth
./run_pipeline.sh --phase3

# 6. Evaluation (opens Jupyter)
#    Interactive notebook for scatter plots and metrics.
./run_pipeline.sh --phase4
```
